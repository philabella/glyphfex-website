Live Excel Sync

Keep an Excel workbook in step with your GlyphFex project so your existing pivots, charts, and formulas keep working as the underlying data refreshes.

What Live Excel Sync does

Most shops have at least one Excel workbook that the office uses for invoicing, customer reports, or year-end summaries. The problem with regular CSV export is that every refresh wipes your pivots and charts — you have to rebuild them.

Live Excel Sync solves this by using a clear-and-rewrite strategy on a multi-sheet .xlsx file. GlyphFex writes data to dedicated sheets; your pivots, charts, and formulas live on different sheets that reference the data sheets. When the data refreshes, the references update and your dashboards keep working.

Live Excel multi-sheet workbook An Excel workbook with auto-generated “Entries”, “Quotes”, “Customers”, and “TimeEntries” data sheets and a user-built “Dashboard” sheet with pivots and charts referencing them.

How to enable Live Excel

  1. Open Live Excel — Tools menu → Live Excel Export, or from the Exports & Reports page.
  2. Pick a target file. Either pick an existing .xlsx file or create a new one. If new, GlyphFex writes 4 starter sheets: Entries, Quotes, Customers, and TimeEntries.
  3. Click Refresh. GlyphFex clears each data sheet and rewrites it with current project data. The first refresh is fast; subsequent refreshes are also fast because we’re only writing data, not regenerating formats.
  4. Build your dashboards in Excel. Create a new sheet (e.g., Dashboard) and use pivots, charts, or formulas that reference cells on the data sheets. Save the workbook.
  5. Refresh whenever the data changes. Re-open Live Excel and click Refresh. Your data sheets update; your dashboard sheet recalculates automatically.

What survives a refresh

The clear-and-rewrite strategy only touches the four data sheets. Anything you add elsewhere in the workbook is preserved:

  • Custom sheets — Dashboards, pivots, charts, and any other sheets you create
  • Formulas referencing data sheets — =SUMIFS(Entries!E:E, Entries!B:B, "Welding") keeps working
  • Pivot tables built on data sheets — Refresh the pivot inside Excel (or set it to refresh on open) and it picks up the new rows
  • Charts — Chart series ranges update automatically as data sheets grow or shrink
  • Cell formatting on data sheets — column widths, header colors, and frozen panes are restored on refresh; cell-level formatting on data rows is not preserved (data rows are rewritten)
What is NOT preserved on the data sheets Anything you type into rows on the data sheets gets wiped on refresh — that’s the price of keeping them in step with the source. Put your additions on a different sheet that references the data sheet via formula.

The four data sheets

EntriesJob in the projectMaster list of jobs with all built-in and custom fields

CustomersUnique customer name across all entriesCustomer summary — total jobs, total value, last activity

Columns on each sheet exactly mirror the corresponding GlyphFex tables. Column headers use snake_case (customer_name, due_date, actual_hours) so they match the field names used in the rest of the product.

The recommended workflow

  1. Set up your data sheets once by clicking Refresh on a fresh workbook.
  2. Spend an hour building the office’s dashboards on a new Dashboard sheet using pivots, slicers, and charts.
  3. Save the workbook to a shared network folder (e.g., \\fileserver\Reports\GlyphFex.xlsx).
  4. Whoever runs end-of-day reports opens the workbook and clicks Refresh in GlyphFex when they want fresh numbers.
  5. If you need a fully scheduled, hands-off version of this, use the Data Feed instead — it drops CSV/JSON files on a folder every N minutes and Power Query / Power BI can pull from there with no manual click required.

When to use Live Excel vs. the alternatives

ToolBest forTrade-off
Live Excel Sync You want Excel as your single source for office reporting and you’re willing to click Refresh manually Manual refresh; one workbook at a time
Data Feed You want unattended, scheduled drops into Power BI, an ERP, or an accountant’s folder Stable file paths only — not a one-click pivot setup
One-shot CSV / Excel export You need to email a snapshot for a single meeting or to a tax accountant Snapshot only — no live refresh

Gotchas

The file is locked

If you have the workbook open in Excel while GlyphFex tries to refresh, the write fails and GlyphFex shows a toast telling you the file is in use. Close Excel (or just close the workbook) and click Refresh again.

I lost data on a data sheet

If you typed into a row on Entries, Quotes, Customers, or TimeEntries and then clicked Refresh, the typed data is gone. The data sheets are derived from GlyphFex — do your hand-edits on a separate sheet, or do them inside GlyphFex itself so they survive the refresh.

My pivot table didn’t update

Excel pivot tables don’t auto-refresh by default. Right-click the pivot and pick Refresh, or set Pivot Options → Data → Refresh data when opening the file. Charts based on the same source will then update automatically.

I want a different set of columns

The column set is fixed by GlyphFex to keep the schema stable for your formulas. If you need a custom column, derive it on your Dashboard sheet using a formula that references the data sheet. The same stability promise applies to columns as to the Data Feed v1 schema — we won’t remove or rename existing columns; new columns appear at the right.
